The problem

Cognition's SWE-bench technical report documents a precise grading pipeline: strip git remotes, reset test files, apply agent patch then test patch. Their SWE-1.6 research update notes real eval fragility — dependency issues, inconsistent timeouts, patch collection edge cases. Most teams publish scores without preflighting those invariants.

Seven doctor gates

  1. test_file_integrity — test files from test_patch reset before extraction
  2. patch_ordering — agent patch before test_patch
  3. git_leakage — no remote / no extra commits
  4. timeout_policy — run within profile ceiling (45m SWE-bench)
  5. grading_env_fingerprint — dependency manifest + lock hash
  6. patch_apply_dry_run — patch applies cleanly
  7. fail_to_pass_manifest — expected tests non-empty
